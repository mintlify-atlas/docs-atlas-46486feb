---
title: gitagent info
description: Display a formatted summary of an agent's configuration.
---

`gitagent info` reads `agent.yaml` and the surrounding directory structure, then prints a human-readable summary of every important field — identity, model, skills, tools, sub-agents, runtime, compliance, and a SOUL.md preview.

## Synopsis

```bash
gitagent info [options]
```

## Options

| Option | Default | Description |
|--------|---------|-------------|
| `-d, --dir <dir>` | `.` | Agent directory to inspect |

## What it displays

| Section | Source | Details |
|---------|--------|---------|
| Name and version | `agent.yaml` | `name` and `version` as the heading |
| Description | `agent.yaml` | `description` field |
| Author | `agent.yaml` | `author` field, if set |
| License | `agent.yaml` | `license` field, if set |
| Model | `agent.yaml` | `preferred`, `fallback` models, and all `constraints` (temperature, max_tokens, etc.) |
| Skills | `skills/` directory | Names of all subdirectories found in `skills/` |
| Tools | `tools/` directory | Names of all `*.yaml` files found in `tools/` (without extension) |
| Sub-agents | `agents/` directory | Names of all subdirectories and `*.md` files found in `agents/` |
| Runtime | `agent.yaml` | `max_turns`, `temperature`, `timeout` |
| Compliance | `agent.yaml` | `risk_tier`, frameworks, `human_in_the_loop`, supervisor, audit logging, retention period, model inventory ID, validation cadence, PII handling, data classification |
| Tags | `agent.yaml` | `tags` list |
| Soul preview | `SOUL.md` | First five lines of the file |

## Examples

```bash
# Inspect the agent in the current directory
gitagent info

# Inspect a specific agent directory
gitagent info -d ./examples/standard

# Inspect after cloning
git clone https://github.com/user/my-agent && gitagent info -d ./my-agent
```
