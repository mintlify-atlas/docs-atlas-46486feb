title: gitagent init
description: Scaffold a new agent repository from a template.
---

`gitagent init` creates the directory structure and starter files for a new gitagent repository. Choose a template based on how much scaffolding you need.

## Synopsis

```bash
gitagent init [options]
```

## Options

| Option | Default | Description |
|--------|---------|-------------|
| `-t, --template <name>` | `standard` | Template to use: `minimal`, `standard`, or `full` |
| `-d, --dir <dir>` | `.` | Target directory to scaffold into |

<Note>
If `agent.yaml` already exists in the target directory, `init` exits with an error to prevent overwriting an existing agent.
</Note>

## Templates

| Template | Best for | Files created |
|----------|----------|---------------|
| `minimal` | Prototypes, embedded agents, learning the spec | `agent.yaml`, `SOUL.md` |
| `standard` | Most agents вЂ” includes skills and knowledge scaffolding | `agent.yaml`, `SOUL.md`, `RULES.md`, `AGENTS.md`, `skills/example-skill/SKILL.md`, `knowledge/index.yaml` |
| `full` | Production agents with compliance, hooks, and memory | Everything in `standard` + `DUTIES.md`, `memory/`, `hooks/`, `workflows/`, `examples/`, `agents/`, `compliance/`, `config/`, `.gitignore` |

## Examples

```bash
# Scaffold a standard agent in the current directory
gitagent init

# Create a minimal agent in a new subdirectory
gitagent init --template minimal --dir ./my-agent

# Create a full production agent
gitagent init --template full --dir ./compliance-agent

# Shorthand flags
gitagent init -t standard -d ./my-agent
```
