---
title: gitagent skills
description: Search, install, list, and inspect agent skills from registries or local directories.
---

`gitagent skills` is a collection of subcommands for managing the skills available to your agent. Skills follow the [Agent Skills](https://agentskills.io) standard.

## Subcommands

| Subcommand | Description |
|------------|-------------|
| `skills search <query>` | Search a registry for skills matching a query |
| `skills install <name>` | Install a skill from a registry or local path |
| `skills list` | List all discovered skills for an agent |
| `skills info <name>` | Show detailed information about a skill |

---

## skills search

Search a skill registry for skills matching a query string.

```bash
gitagent skills search <query> [options]
```

### Options

| Option | Default | Description |
|--------|---------|-------------|
| `-p, --provider <name>` | `skillsmp` | Registry provider: `skillsmp` or `github` |
| `-d, --dir <dir>` | `.` | Agent directory (used to read registry config from `agent.yaml`) |
| `-l, --limit <n>` | `20` | Maximum number of results to return |

### Examples

```bash
# Search the default SkillsMP registry
gitagent skills search "code review"

# Search GitHub, limit to 5 results
gitagent skills search "pdf reader" --provider github --limit 5

# Search using the registry configured in agent.yaml
gitagent skills search "data analysis" -d ./my-agent
```

## skills install

Install a skill from a registry or local path into the agent's `skills/` directory (or globally).

```bash
gitagent skills install <name> [options]
```

### Options

| Option | Default | Description |
|--------|---------|-------------|
| `-p, --provider <name>` | `skillsmp` | Registry provider: `skillsmp`, `github`, or `local` |
| `-g, --global` | `false` | Install to `~/.agents/skills/` instead of the agent's `skills/` directory |
| `-d, --dir <dir>` | `.` | Agent directory for local installs |

### GitHub format

When using `--provider github`, the skill name is specified as `owner/repo#path/to/skill`:

```bash
gitagent skills install anthropics/skills#skills/code-review --provider github
```

### Examples

```bash
# Install from SkillsMP into the current agent
gitagent skills install code-review

# Install globally (available to all agents)
gitagent skills install code-review --global

# Install from GitHub
gitagent skills install owner/repo#skills/code-review --provider github

# Install a local skill directory
gitagent skills install ./my-custom-skill --provider local

# Install into a specific agent
gitagent skills install code-review -d ./my-agent
```

## skills list

Discover and list all skills available to an agent. Skills are found by searching multiple paths in priority order.

```bash
gitagent skills list [options]
```

### Options

| Option | Default | Description |
|--------|---------|-------------|
| `-d, --dir <dir>` | `.` | Agent directory |
| `-l, --local` | `false` | Only show skills in the agent's own `skills/` directory |

### Skill discovery paths

Skills are discovered from these locations (highest to lowest priority):

| Priority | Path | Source |
|----------|------|--------|
| 1 | `<agent>/skills/` | Agent-local |
| 2 | `<agent>/.agents/skills/` | agentskills.io standard |
| 3 | `<agent>/.claude/skills/` | Claude Code |
| 4 | `<agent>/.github/skills/` | GitHub |
| 5 | `~/.agents/skills/` | Personal (global) |

The first match by name wins (deduplication).

### Examples

```bash
# List all discovered skills
gitagent skills list

# List only agent-local skills
gitagent skills list --local

# List skills for a specific agent
gitagent skills list -d ./my-agent
```

### Example output

```
Discovered Skills
────────────────────────────
  code-review
    Thorough code review with security analysis
    source: agent-local (/path/to/my-agent/skills/code-review)
    license: MIT

  security-audit
    Security audit skill for dependency and config checks
    source: global (/home/user/.agents/skills/security-audit)
    license: Apache-2.0

2 skills found
```

## skills info

Show detailed information about a specific skill, including its frontmatter, allowed tools, metadata, and an instruction preview.

```bash
gitagent skills info <name> [options]
```

### Options

| Option | Default | Description |
|--------|---------|-------------|
| `-d, --dir <dir>` | `.` | Agent directory used for skill discovery |

### Examples

```bash
# Inspect a skill by name
gitagent skills info code-review

# Inspect a skill in a specific agent
gitagent skills info code-review -d ./my-agent
```

## Registry providers

| Provider | Search | Install |
|----------|--------|---------|
| `skillsmp` | SkillsMP REST API (`https://api.skillsmp.com`) | Download and extract archive |
| `github` | GitHub code search | Sparse clone from `owner/repo#path` |
| `local` | *(not searchable)* | Copy from local filesystem path |

<Tip>
You can configure a default registry in `agent.yaml` under a `registries` field. If present, `skills search` and `skills install` use the first configured registry unless overridden with `--provider`.
</Tip>
