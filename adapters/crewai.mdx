---
title: "CrewAI adapter"
description: "Export a gitagent agent as a CrewAI YAML configuration and run it with crewai kickoff."
---

The `crewai` adapter converts your gitagent directory to a [CrewAI](https://crewai.com) crew configuration in YAML, writes it to a temporary file, and runs it with `crewai kickoff --config <tmpfile>`. The temp file is deleted after the process exits.

## Requirements

- `crewai` CLI installed (`pip install crewai`)

## How it works

<Steps>
  <Step title="Parse SOUL.md">
    The adapter parses `SOUL.md` into sections by heading. The `Core Identity` section becomes the agent's `role`, and `Values & Principles` becomes the `goal`. If those sections are absent, both fall back to the `description` field in `agent.yaml`.
  </Step>
  <Step title="Build backstory">
    The backstory is the full `SOUL.md` text with headings stripped, trimmed to 2000 characters. Skill descriptions are appended as a capabilities list.
  </Step>
  <Step title="Add sub-agents">
    Each entry under `agents/` in `agent.yaml` becomes a separate CrewAI agent in the YAML. The sub-agent's `SOUL.md` is used as its backstory if it exists.
  </Step>
  <Step title="Run kickoff">
    The YAML is written to a temp file and passed to `crewai kickoff --config <tmpfile>` with `stdio: 'inherit'`.
  </Step>
</Steps>

## Generated YAML structure

The adapter produces a YAML file in this shape:

```yaml generated crew config
# CrewAI configuration generated by gitagent export
# Source: code-review-agent v1.0.0

agents:
  code-review-agent:
    role: >-
      Senior code review specialist with expertise in security analysis and
      best-practice enforcement
    goal: >-
      Deliver objective, actionable code reviews that improve quality
      and prevent vulnerabilities
    backstory: >-
      Experienced engineer focused on quality and security...

      Capabilities (Skills):
      - code-review: Performs thorough code reviews with security analysis
    verbose: true
    allow_delegation: false
```

### SOUL.md section extraction

The adapter looks for these heading patterns in `SOUL.md`:

| SOUL.md heading | CrewAI field | Max length |
|----------------|--------------|------------|
| `## Core Identity` | `role` | 500 chars |
| `## Values & Principles` | `goal` | 500 chars |
| Full content (headings stripped) | `backstory` | 2000 chars |

### Sub-agent mapping

For each entry in the `agents` section of `agent.yaml`, the adapter creates a corresponding CrewAI agent:

```yaml agent.yaml sub-agent
agents:
  fact-checker:
    description: Verifies factual claims before publication
```

```yaml generated sub-agent
agents:
  fact-checker:
    role: Verifies factual claims before publication
    goal: Verifies factual claims before publication
    backstory: ''   # populated from agents/fact-checker/SOUL.md if it exists
    verbose: true
```

Delegation (`allow_delegation`) is set to `true` on the main agent only when `delegation.mode: auto` is set in `agent.yaml`.

## Export format

You can inspect the generated YAML without running it:

```bash
# Write YAML to a file
gitagent export -f crewai -d ./my-agent -o crew.yaml

# Print to stdout
gitagent export -f crewai -d ./my-agent
```

## Examples

<CodeGroup>

```bash basic run
gitagent run -d ./my-agent -a crewai
```

```bash from git repo
gitagent run -r https://github.com/user/my-agent -a crewai
```

```bash export and run manually
gitagent export -f crewai -d ./my-agent -o crew.yaml
crewai kickoff --config crew.yaml
```

</CodeGroup>

<Note>
The CrewAI adapter does not pass a prompt or task description at run time. CrewAI reads task definitions from its own configuration. If you need to parameterize the task, export the YAML and edit it before running `crewai kickoff`.
</Note>
