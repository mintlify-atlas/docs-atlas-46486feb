---
title: "OpenAI adapter"
description: "Export a gitagent agent as Python code using the OpenAI Agents SDK, then run it with python3."
---

The `openai` adapter generates Python source code that instantiates your agent using the [OpenAI Agents SDK](https://openai.github.io/openai-agents-python/), writes it to a temporary file, and executes it with `python3`. The temp file is deleted after the process exits.

## Requirements

- Python 3 installed and on your `PATH`
- `openai-agents` package installed (`pip install openai-agents`)
- `OPENAI_API_KEY` environment variable set

## How it works

<Steps>
  <Step title="Check credentials">
    The runner verifies that `OPENAI_API_KEY` is set. If it is not, the process exits with an error.
  </Step>
  <Step title="Generate Python code">
    The adapter calls `exportToOpenAI`, which produces a complete Python module. The module imports `Agent` and `Tool` from the `agents` package, defines a Python function stub for each tool in `tools/*.yaml`, and instantiates an `Agent` with the name, instructions, model, and tools list.
  </Step>
  <Step title="Write and execute">
    The generated code is written to a temp `.py` file under the system temp directory. `python3` is spawned with `stdio: 'inherit'` and the agent directory as the working directory.
  </Step>
  <Step title="Clean up">
    The temp file is deleted after `python3` exits, regardless of exit status.
  </Step>
</Steps>

## Generated Python code

The adapter produces code in this shape:

```python generated agent.py
"""
OpenAI Agents SDK definition for my-agent v1.0.0
Generated by gitagent export
"""

from agents import Agent, Tool

def lint_check(target: str, rules: str = None):
    """Run lint checks on the specified target"""
    # TODO: Implement tool logic
    pass

def complexity_analysis(file_path: str):
    """Analyze code complexity for a file"""
    # TODO: Implement tool logic
    pass

my_agent_agent = Agent(
    name="my-agent",
    instructions="""# my-agent v1.0.0
...""",
    model="claude-sonnet-4-5-20250929",
    tools=[lint_check, complexity_analysis],
)
```

### Tool generation

For each YAML file in `tools/`, the adapter reads the `input_schema` and maps JSON Schema types to Python type hints:

| JSON Schema type | Python type |
|-----------------|-------------|
| `string` | `str` |
| `integer` | `int` |
| `number` | `float` |
| `boolean` | `bool` |
| `array` | `list` |
| `object` | `dict` |

Required parameters are typed without a default. Optional parameters receive `= None`.

Hyphens in tool names are converted to underscores to produce valid Python identifiers.

### System prompt

The `instructions` field contains the assembled system prompt: `SOUL.md`, `RULES.md`, skills with their allowed-tools, an `agents/openai.yaml` MCP config if it exists, and compliance constraints.

## Export format

You can generate the Python file without executing it:

```bash
# Write Python code to a file
gitagent export -f openai -d ./my-agent -o agent.py

# Print to stdout
gitagent export -f openai -d ./my-agent
```

## Examples

<CodeGroup>

```bash basic run
export OPENAI_API_KEY="sk-..."
gitagent run -d ./my-agent -a openai
```

```bash from git repo
export OPENAI_API_KEY="sk-..."
gitagent run -r https://github.com/user/my-agent -a openai
```

```bash export only
gitagent export -f openai -d ./my-agent -o agent.py
python3 agent.py
```

</CodeGroup>

<Note>
The generated Python file contains function stubs with `pass` bodies. The agent will run, but tools that require real implementations need to be filled in manually. Use `gitagent export -f openai` to get the file, implement the tool functions, then run it directly with `python3`.
</Note>
