Adapters are the bridge between a gitagent directory and a running AI framework. Define your agent once in `agent.yaml` and `SOUL.md`; adapters handle translating that definition into whatever format the target framework expects.

## Two ways to use adapters

Adapters are exposed through two commands:

| Command | What it does |
|---------|-------------|
| `gitagent export -f <format>` | Converts the agent to a framework-specific file format and writes it to stdout or a file. No execution happens. |
| `gitagent run -a <adapter>` | Runs the agent end-to-end â€” exports the format, writes any temp files, and launches the framework process. |

Use `export` when you want to inspect the generated output, commit it to your repo, or pass it to a CI pipeline. Use `run` when you want to execute the agent immediately.

```bash
# Export only â€” inspect the output
gitagent export -f openai -d ./my-agent -o agent.py

# Run end-to-end
gitagent run -d ./my-agent -a openai -p "Review this code"
```

## All adapters

| Adapter | `run` mode | Prompt required | Requirements |
|---------|------------|-----------------|-------------|
| `claude` | Interactive or one-shot | No (optional) | [Claude Code CLI](https://docs.anthropic.com/en/docs/claude-code) |
| `openai` | One-shot | No | Python 3, `openai-agents` package, `OPENAI_API_KEY` |
| `crewai` | One-shot | No | `crewai` CLI (`pip install crewai`) |
| `openclaw` | One-shot | **Yes** (`-p`) | OpenClaw CLI, `ANTHROPIC_API_KEY` |
| `nanobot` | Interactive or one-shot | No (optional) | Nanobot CLI (`pip install nanobot-ai`), `ANTHROPIC_API_KEY` |
| `lyzr` | One-shot | **Yes** (`-p`) | `LYZR_API_KEY` |
| `github` | One-shot | **Yes** (`-p`) | `GITHUB_TOKEN` or `GH_TOKEN` with `models:read` scope |
| `git` | Auto-detect | Depends on detected adapter | Depends on detected adapter |
| `prompt` | Print only | No | None |

## How to choose an adapter

**You already use Claude Code** â€” Use the `claude` adapter. It gives you an interactive terminal session with full tool access, file permissions, and hook support.

**You need OpenAI models or the Agents SDK** â€” Use the `openai` adapter. It generates Python code using the OpenAI Agents SDK and runs it with `python3`.

**You're on a CrewAI team** â€” Use the `crewai` adapter. It exports a YAML crew configuration and runs `crewai kickoff`.

**You want a quick one-shot answer without managing credentials yourself** â€” Use the `github` adapter. GitHub Models is accessible with any GitHub token that has the `models:read` scope, which many developers already have.

**You want to deploy to a hosted platform** â€” Use the `lyzr` adapter. It creates a persistent agent on Lyzr Studio that you can chat with, update, and share.

**You're cloning an unknown repo and want it to just work** â€” Use `git`. The git runner auto-detects the best adapter from the repo's contents and delegates to it.

**You need just the system prompt text** â€” Use `prompt`. It prints the assembled system prompt to stdout and exits without launching any framework.
